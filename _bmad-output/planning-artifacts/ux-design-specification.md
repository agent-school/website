# UX Design Specification: Agent School Website

**Author:** Jack Luo  
**Date:** February 12, 2026  
**Design Era:** Elegant Minimal Print (2026) — Post-Cyber Citrus  
**Research Foundation:** Stripe + Linear + Editorial Design Patterns

---

## Executive Summary

This specification defines a **completely unique visual identity** for Agent School's marketing website. Departing from the Cyber Citrus gradient aesthetic, this design creates an **editorial luxury system** that feels between futuristic precision and timeless elegance.

**Design Philosophy:** *"Financial Times meets Linear's FIG system meets Stripe's data-driven minimalism."*

### Core Principles

1. **Editorial Structure** — Every element is numbered/labeled like a magazine (FIG, CASE, STEP)
2. **Typographic Rhythm** — Alternating roman/italic creates visual cadence
3. **Data as Design** — Numbers and metrics integrated into visual hierarchy
4. **Signature Motif** — Hollow circle (○) repeated throughout as Agent School's parallelogram
5. **Mathematical Precision** — Golden ratio spacing (1.618), strict grid alignment
6. **Print-to-Digital** — Bringing editorial magazine design to web

---

## Research-Backed Design Decisions

### What Makes This Unique?

After studying Stripe, Linear, and award-winning editorial sites, I've identified **10 distinctive patterns** that will make Agent School stand out:

1. **FIG Numbering System** (Linear-inspired) — "FIG 1.0", "FIG 2.3", "CASE 001"
2. **Hollow Circle Motif** (Stripe-inspired) — ○ as signature element throughout
3. **Data Integration** — Stats not decorative, but structural design elements
4. **Italic Emphasis** — Strategic use of italic caps for rhythm (Stripe pattern)
5. **Hairline Grid** — 1px rules create editorial magazine feel
6. **Monochrome + Sage** — 95% grayscale, 5% soft sage accents
7. **Golden Ratio Spacing** — 1.618 multiplier for vertical rhythm
8. **Typographic Contrast** — Didone serif + Grotesk sans (extreme contrast)
9. **Timeline Visualizations** — Horizontal phase bars (Linear-inspired)
10. **Micro-Typography** — Kerning, tracking, leading meticulously refined

**Competitive Differentiation:**
- Most SaaS sites: Bright gradients, rounded cards, playful → Agent School: Monochrome, flat, editorial
- Most AI sites: Neon purples, glossy effects → Agent School: Sage accents, matte finish
- Most automation sites: Busy illustrations → Agent School: Clean diagrams with hairline rules

## Signature Visual System

### The Hollow Circle (○)

Agent School's **signature motif** — the hollow circle — appears consistently throughout:

**Navigation:** Far-left indicator (○ agent school)  
**Section Markers:** Numbered circles (○ FIG 1.0, ○ FIG 2.3)  
**Step Indicators:** Progress circles (○●○ = Step 2 of 3)  
**Diagrams:** Node points in workflow visualizations  
**Stats:** Bullet points in metric cards (○ 99% Accuracy)

**Visual DNA:**
```
○ = Potential (empty circle waiting to be filled with agent knowledge)
● = Realized (filled circle representing certified agent)
◐ = In Progress (half-filled = learning phase)
```

This creates a **visual thread** connecting all elements — subtle but distinctive.

## Typography System

### Font Selections (Free & Licensed Options)

**Display Serif (Headlines):**
- **Recommended:** Libre Bodoni (free, Google Fonts)
- **Premium Alternative:** Bodoni Moda (slightly refined)
- **Characteristics:** High contrast, thin hairlines, thick verticals, sharp terminals
- **Usage:** All headlines, numbers in stats, "FIG" labels

**UI Sans (Everything Else):**
- **Recommended:** Inter (free, Google Fonts)
- **Premium Alternative:** Neue Haas Grotesk (if budget allows)
- **Characteristics:** Neutral, clean, geometric, excellent legibility
- **Weights:** Light (300), Regular (400), Medium (500)
- **Usage:** Body copy, navigation, labels, captions

**Monospace (Diagrams Only):**
- **Recommended:** JetBrains Mono (free)
- **Characteristics:** Code-optimized, clean, functional
- **Usage:** Only inside diagram code capsules (e.g., "DETECT: legacy_button_042")

**Purpose:** Add dimensional depth without breaking monochrome

```css

background: radial-gradient(
  circle at 20% 30%, 
  rgba(193, 213, 168, 0.03), 
  transparent 60%
);

/* Methodology section */

  circle at 80% 70%, 
  rgba(168, 198, 134, 0.02), 
  transparent 60%

```

**Rules:**
- Opacity: 2-3% maximum
- Placement: Off-center (20%/80% horizontal, 30%/70% vertical)
- Radius: 60%+ fade
- Color: Only sage tones, never multiple colors

## Mathematical Grid System

### Golden Ratio Spacing (1.618)

Instead of arbitrary 8px increments, use **golden ratio multipliers** for vertical rhythm:

**Base Unit:** 8px

**Spacing Scale (Fibonacci-inspired):**
```
XS:   8px   (1 × 8)
S:    13px  (1.618 × 8, rounded)
M:    21px  (1.618 × 13)
L:    34px  (1.618 × 21)
XL:   55px  (1.618 × 34)
XXL:  89px  (1.618 × 55)
XXXL: 144px (1.618 × 89)
```

**Application:**
- Margin between body paragraphs: M (21px)
- Padding inside cards: L (34px)
- Section vertical spacing: XXL (89px)
- Hero section padding top: XXXL (144px)

### 12-Column Grid

**Container Max-Width:** 1200px (content), 1400px (full-width sections)  
**Column Width:** Fluid (calculated by CSS Grid)  
**Gutter:** 24px (3 × base unit)

**Breakpoints:**
```
Mobile:  320px - 767px  (4 columns)
Tablet:  768px - 1023px (8 columns)
Desktop: 1024px+        (12 columns)
```

**Layout Patterns:**

**Hero Section:**
```
[---- 8 columns ----][-- 4 col --]
     Content          Stats Band
```

**Methodology:**
```
[-- 4 col --][-------- 8 columns --------]
  Step Info      Diagram/Visual
```

**Consultation Form:**
```
[------ 6 columns ------][------ 6 columns ------]
       Left Panel               Right Panel
```

## Section Labeling System (Simplified)

Agent School uses **clean section kickers** for structure without over-labeling:

### Hierarchy

**CASE [Number]** — Case studies (001, 002, etc.)  
**STEP [Number]** — Process steps (01, 02, 03)  
**PHASE [Number]** — Timeline phases (01, 02, 03)

### Examples

**Homepage Sections:**
```
Hero Section
├─ CASE STUDY 001 — Hotel Operations Automation
└─ Stats Band (99% accuracy, 14d training, etc.)

Methodology Section
├─ HOW WE BUILD TRUST (kicker)
├─ Headline + Subhead
└─ PHASE 01 — Environmental Mapping (timeline)

Process Steps
├─ STEP 01: CARTOGRAPHY — Shadow Discovery
├─ STEP 02: PEDAGOGY — Teaching Interface
└─ STEP 03: VALIDATION — Certification Testing
```

**Visual Treatment:**

```
CASE STUDY 001 — HOTEL OPERATIONS AUTOMATION
↑ Small uppercase kicker, Inter Medium, 11px, 0.12em tracking
  Color: Charcoal 700
```

This creates **editorial clarity** without over-numbering — cleaner, more focused.

## Animation Principles (Refined)

### Micro-Interactions (Subtle)

**Button Hover:**
```
Default → Hover:
- Background: Transparent → Sage 100
- Border: Charcoal 900 → Sage 300
- Duration: 200ms
- Easing: ease-out
```

**Link Hover:**
```
Default → Hover:
- Underline: None → 1px solid
- Color: Charcoal 700 → Charcoal 900
- Duration: 150ms
- Easing: ease-out
```

**Input Focus:**
```
Default → Focus:
- Border-bottom: 1px Hairline → 2px Sage 300
- Duration: 200ms
- Easing: ease-out
```

### Scroll Animations (Fade + Drift)

**Pattern:**
```
Initial state:
- Opacity: 0
- Transform: translateY(20px)

Final state:
- Opacity: 1
- Transform: translateY(0)

Animation:
- Duration: 800ms
- Easing: cubic-bezier(0.25, 0.1, 0.25, 1)
- Trigger: Element enters viewport (with 80px margin)
```

**Stagger (for lists/grids):**
```
Child 1: Delay 0ms
Child 2: Delay 100ms
Child 3: Delay 200ms
(etc.)
```

**What NOT to animate:**
- ❌ No parallax scrolling
- ❌ No bouncing effects
- ❌ No rotation
- ❌ No scale transforms (except button scale: 0.98 on click)
- ❌ No sliding panels
- ❌ No revealing text character-by-character

**Philosophy:** Animations should feel like *natural reveals*, not *choreographed performances*.

## Accessibility (WCAG AA Compliant)

### Color Contrast Ratios

**Text on Off-white (#FAFAFA):**
- Charcoal 900 (#2A2A2A): **14.2:1** ✅ (exceeds AAA)
- Charcoal 700 (#4A4A4A): **8.9:1** ✅ (exceeds AAA)
- Charcoal 500 (#6A6A6A): **5.2:1** ✅ (meets AA)
- Charcoal 300 (#9A9A9A): **3.1:1** ⚠️ (decorative only, not body text)

**Sage Accents:**
- Sage 400 (#A8C686) on Off-white: **3.2:1** (UI elements only, not text)
- Sage 500 (#8FB570) on Off-white: **4.2:1** (meets AA for large text)

**Never use:**
- Charcoal 300 for body text (fails AA)
- Sage 100-300 for text (too low contrast)

### Focus Indicators

**All interactive elements:**
```
Default: No visible outline
Focus: 2px solid Sage 300 outline
        Offset: 2px
        Border-radius: 0
```

**Keyboard navigation:**
- Tab order: Logical top-to-bottom, left-to-right
- Skip link: "Skip to main content" (visually hidden until focused)
- Focus trap: Modal/overlay traps focus until closed

### Screen Readers

**Semantic HTML:**
```html
<header>
  <nav aria-label="Main navigation">
<main>
  <section aria-labelledby="methodology-heading">
    <h2 id="methodology-heading">
<aside aria-label="Contact form">
<footer>
```

**ARIA Labels:**
- Hollow circles: `aria-hidden="true"` (decorative)
- Icons: `aria-label="Right arrow"` (if meaningful)
- Diagrams: `<figure>` with `<figcaption>` or `aria-describedby`

**Alt Text:**
- Decorative elements: `alt=""` (empty string)
- Informative images: Descriptive alt text (e.g., "Workflow diagram showing data flow from detection to semantic understanding")
